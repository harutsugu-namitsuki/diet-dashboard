# ダイエット・モチベーションダッシュボード 開発計画

ご要望ありがとうございます。ITコンサルタントとして、毎日見るだけでモチベーションが「ぶち上がる」最高にクールで実用的なSPA（シングルページアプリケーション）を設計します。

GitHubのプライベートリポジトリからPagesを通じて配信（またはローカルで起動）でき、バックエンド不要でブラウザ内にデータを保存するセキュアな設計とします。

## 概要とコンセプト
* **コンセプト**: 「行動を称賛し、明日への活力を生むコックピット」
* **アーキテクチャ**: React (Vite) + Vanilla CSS (プレミアムなダークモード＆グラスモーフィズムデザイン) + LocalStorage (データ保存) + Recharts (美麗なグラフ)
* **テイスト**: サイバーパンクやスポーティな印象を与える流線型のデザイン、滑らかなマイクロアニメーション、目標達成時のリワード（パーティクル等）。

> [!NOTE]
> GitHub Pages環境（静的サイト）で動作させるため、複雑なデータベースは使わずブラウザの `localStorage` に記録を蓄積する設計とします。これによりユーザー（あなた）だけがアクセスでき、維持費もゼロで済みます。

## アプリケーション機能要件

1. **ヒーローステータスボード**
   * **現在の体重** と **目標 (73.1kg) までの残り** を大きくハイライト。
   * **本日の目標体重**: スタート（85.0kg）からゴールまでの理想的な下降線を計算し、「今日は〇〇kg以下ならオンペース！」という指標を毎日算出。
   * **残り日数カウントダウン**: 84日間のうち、現在何日目か・残り何日かをプログレスバーで視覚化。

2. **インタラクティブ・チャート（進捗グラフ）**
   * X軸：日付、Y軸：体重
   * **理想ライン（Target Trend）** と **実績ライン（Actual Weight）** の2軸を表示し、自分が今オンスケジュールかどうかが一目でわかるようにします。

3. **デイリー入力フォーム**
   * 毎日の体重、体脂肪率などを入力するフォーム。
   * 入力すると即座にグラフとステータスボードがアニメーション付きで更新されます。

4. **モチベーション・エンジン**
   * ログイン時や体重更新時に、前日比でマイナスだった場合に褒め称えるアニメーションやダイアログを表示。

## 提案する技術スタックと構成

### 変更・作成されるコンポーネント
1. **基盤環境の構築**
   * `npx create-vite-app@latest` を用いてReactプロジェクトを初期化。
2. **ルーティング不要のSPA構成**
   * `App.jsx`: 全体のレイアウトとステート（状態）管理、LocalStorageとの同期。
3. **コンポーネント群**
   * `DashboardCard.jsx`: 各種KPI（本日の目標値など）を表示するパネル。
   * `ProgressChart.jsx`: Rechartsを利用したグラフコンポーネント。
   * `InputForm.jsx`: データ入力欄。
4. **スタイリング (`index.css`)**
   * Tailwind等のフレームワークは避け、Vanilla CSSで「プレミアムでモダンなデザイン（グラデーション、シャドウ、ホバーエフェクト）」をゼロから構築します。

## ユーザーレビューのお願い (Open Questions)

> [!IMPORTANT]
> 開発を開始する前に、以下の点について方針のすり合わせをお願いします。

1. **デザインの方向性（カラーパレット）**: 
   テンションが上がるデザインとして、「サイバーなネオンカラー（黒背景×ブルー/パープル）」や「スポーティなビタミンカラー（ダークグレー×オレンジ/イエロー）」などを想定しています。お好みのテイストはありますか？
2. **追加したい項目**: 
   体重の他に、入力・グラフ化したい項目（例：体脂肪率、カロリー赤字の実績値など）はありますか？まずはシンプルに「体重」に特化する形でも構いません。

## 検証プラン (Verification Plan)
1. **ローカル開発環境での動作確認**
   - `npm run dev`で起動し、初期データ（25日目までの実績）が正しくグラフ化・表示されるかを検証。
2. **データ入力テスト**
   - フォームから新しい体重を入力し、LocalStorageに保存され、リロードしてもデータが消えないことを確認。
3. **UI/UXレビュー**
   - アニメーションやデザインの質感が「プレミアム名仕上がり」になっているかを確認。

問題なければ、この計画の承認をお願いいたします。（また、デザインの色味などのご希望があれば教えてください！）
